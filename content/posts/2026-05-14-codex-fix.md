[解决链接](https://github.com/openai/codex/issues/14209#issuecomment-4095920456)

**问题**：openai访问必须走代理，但是codex并不能稳定命中代理，且没有提供代理设置，，因而第一次需要很长时间来建立连接；如果强制设置`HTTP_PROXY`和`HTTPS_PROXY`，会导致某些app（如workbuddy）只能在开启代理时才能使用。

**解决**：新建一个批处理文件如：`start-codex-proxy.bat`，内容如下
```
@echo off

set HTTP_PROXY=http://127.0.0.1:7890
set HTTPS_PROXY=http://127.0.0.1:7890
set http_proxy=http://127.0.0.1:7890
set https_proxy=http://127.0.0.1:7890

start "" "path to Codex.exe"
```
即在当前终端中设置代理环境变量，并启动codex，这样不会影响其他app的正常访问。


**补充**：链接中的解决方案提到要在FLClash中对openai相关的网址进行强制的代理设置，但是实测下来可以不需要。
